# AFGROW Import/Execution Contract

1. Create a new AFGROW model matching `case.json`.
2. Select material data only from the recorded source; archive the exact source and units.
3. Import or enter `spectrum.csv` without rescaling unless the scale is recorded.
4. Save the native project beside this package.
5. Export crack length versus cycles as CSV with columns `cycles,crack_length_m`.
6. Export the run summary and warnings. Do not overwrite the pre-run package.
7. Return the native project, exported CSV, summary, and software version.

This workflow is damage-tolerance screening, not certification substantiation.
